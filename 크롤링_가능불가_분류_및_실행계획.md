## 3. 크롤링 실행 계획 (GFAuction 가능 항목)

### Step 1: 미구현 팝업 3종 파서 추가 개발

현재 `judgement`, `status`, `mun` 3종만 구현됨. 나머지 3종 추가:

| 팝업 | 키 | 수집 데이터 | 예상 작업 |
|------|---|------------|:---:|
| 매각물건명세서 | `mul` | 매각조건, 특별매각조건, 인수사항 상세 | 30분 |
| 부동산의 표시 | `bu` | 공부상 토지/건물 표시 (지번, 지목, 면적 정확값) | 20분 |
| 송달내역 | `song` | 채무자/소유자 송달 성공 여부 (명도 난이도 판단용) | 20분 |

### Step 2: 물건 목록(테이블4) 파서 보강

현재 요약행만 수집 중. 대지권/건물 각 행의 상세 데이터(구조, 용도, 대지권비율, 면적) 분리 저장.

### Step 3: 전체 재크롤링 실행

```
실행 명령: python crawler_incremental.py --force-detail --pages 700
예상 소요: 3~4시간 (33,285건 × 4 HTTP 요청/건 = ~133,000 요청)
```

| 단계 | 내용 | 요청 수 |
|:---:|------|:---:|
| 1 | 메인 상세 페이지 (기존 데이터 + 임차인 + 배당요구종기) | 33,285 |
| 2 | 감정평가서 팝업 (건물스펙 + 본문) | 33,285 |
| 3 | 현황조사서 팝업 | 33,285 |
| 4 | 매각물건명세서 팝업 | 33,285 |
| 5 | 문건접수내역 팝업 | 33,285 |
| 6 | 부동산표시 + 송달내역 팝업 (선택) | 66,570 |

### Step 4: 후처리 배치

```
python risk_tagger.py          # 리스크 키워드 태깅 (5분)
python geocode_batch.py        # 위경도 변환 - 카카오 API 키 필요 (1시간)
python address_splitter.py     # 주소 시/군/구 분리 (신규 개발, 10분)
```

## 5. 작업 순서 요약

```
[즉시 실행 가능]
  1. 미구현 팝업 3종 파서 추가 (mul, bu, song)     ← 1시간
  2. 물건 목록 파서 보강                             ← 30분
  3. 주소 분리 스크립트 개발                          ← 20분
  4. 전체 재크롤링 실행                              ← 3~4시간 (백그라운드)
  5. 리스크 태깅 배치                                ← 5분

[API 키 발급 후]
  6. 카카오 API 키 → 위경도 변환                     ← 1시간
  7. 공공데이터 API 키 → 실거래가 연동               ← 3시간
  8. 건축물대장 API 연동                             ← 2시간
```

**즉시 실행하시겠습니까?** Step 1~3 개발 후 Step 4 재크롤링을 바로 시작할 수 있습니다.
